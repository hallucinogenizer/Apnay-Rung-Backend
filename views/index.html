Sign Up

To sign up, you need to send a request to a different URL depending on the type of the user. There is a different URL for signing up a Customer and a different one for signing up a Seller.

Customer Sign Up

URL: https://apnay-rung-api.herokuapp.com/customer/new
REQUEST TYPE: POST

Example Request

POST https://apnay-rung-api.herokuapp.com/customer/new
content-type: application/json

{
"name":"Javed Akhtar",
"email":"javed@gmail.com",
"password":"12345678",
"address":"Pluto"
}

JSON OBJECT

KEY Type Description
name String The full name of the customer.
email String Email address of customer. Must be unique.
password String Make sure the password and re-enter password fields match. Send only one password to the server.
address String Customer's home address.
phone String Customer's phone number.

Responses

HTTP Status Code Outcome Description
201 success Successfully created new user.
500 error Unknown error occured.


Seller Sign Up

URL: https://apnay-rung-api.herokuapp.com/seller/new
REQUEST TYPE: POST

Example

POST https://apnay-rung-api.herokuapp.com/seller/new
content-type: application/json

{ "name":"Maham Ghazanfar",
"email":"maham@gmail.com",
"password":"12345678",
"cnic":"3520212345678",
"cnic_image":"maham_cnic.jpg",
"location":"Kashmir",
"sec_questions":{"What is the name of your cat?":"Ihsan","What is the name of your best friend?":"Ihsan"}
}


JSON OBJECT

KEY Type Description
name String The full name of the customer.
email String Email address of customer. Must be unique.
password String Make sure the password and re-enter password fields match. Send only one password to the server.
cnic String No dashes in CNIC. e.g. 3520123456789.
cnic_image String The file name of the cnic image as returned to you by the server upon upload.
location String Where does the Seller belong to?
sec_questions String A JSON object with string question as key, and its string answer as value.

Responses

HTTP Status Code Outcome Description
201 success Successfully created new user.
500 error Unknown error occured.

Log IN

Login has a different URL for each user type.

Customer Login

URL: https://apnay-rung-api.herokuapp.com/customer/verify
REQUEST TYPE: GET

Example Request

GET https://apnay-rung-api.herokuapp.com/customer/verify
content-type: application/json
{
"email":"example@gmail.com",
"password":"12345678"
}

Seller Login

URL: https://apnay-rung-api.herokuapp.com/seller/verify
REQUEST TYPE: GET

Example Request

GET https://apnay-rung-api.herokuapp.com/seller/verify
content-type: application/json
{
"email":"example@gmail.com",
"password":"12345678"
}

Admin Login

URL: https://apnay-rung-api.herokuapp.com/admin/verify
REQUEST TYPE: GET

Example Request

GET https://apnay-rung-api.herokuapp.com/admin/verify
content-type: application/json
{
"email":"example@gmail.com",
"password":"12345678"
}

JSON OBJECT

KEY Type Description
email String Email address of customer.
password String Must not be empty.

All Users

Only the Admin has the authorization to access the All Users API to retrieve a list of all users of a specific type.

All Customers

Returns a list of all customers and all information (except password) about them.
URL: https://apnay-rung-api.herokuapp.com/customer/all
REQUEST TYPE: GET
